Evoked Agent Governance Architecture

### Excerpt (80 words max)
The Evoked Agent Governance Architecture is a suite of frameworks for governing autonomous AI agent systems. It includes six implementation tools: a Sovereignty Assessment, Restraint Specification builder, Governance Starter Kit, Trust Architecture Blueprint, Agent Voice Architecture Guide, and Agent Memory Architecture Guide. Developed through operational practice governing a 142-agent fleet, these tools address the governance gap in agentic AI - providing structured approaches to agent identity, deliberation integrity, refusal rights, and accountability that no existing framework covers.

### Detailed Description (500 words max)
As organizations deploy autonomous AI agents at scale, a critical governance gap has emerged. Existing frameworks address model-level safety (responsible scaling policies, constitutional AI) and agent-level operations (authentication, lifecycle management, interoperability protocols). None addresses how agents participate in the governance structures that constrain them -how they maintain identity, exercise refusal, or contribute to deliberative decision-making.

The Evoked Agent Governance Architecture fills this gap through six open implementation tools, each addressing a distinct governance dimension:

**Sovereignty Assessment Toolkit** -A 47-point diagnostic across seven domains (identity, memory, governance, boundaries, voice, community, evolution) that evaluates whether an AI system honors or exploits its users. Produces a domain health map and prioritized action plan.

**Agent Restraint Specification** -A framework for defining what AI agents must refuse, how they refuse it, and how organizations verify refusal fidelity. Treats refusal as a governance right exercised by agents, not merely a constraint imposed on them.

**Agent Governance Starter Kit** -Charter, boundaries, drift thresholds, accountability structures, and authority frameworks for organizations deploying multi-agent systems. Addresses the gap between single-model safety and fleet-level governance.

**Trust Architecture Blueprint** -Four-pillar framework (identity, memory, governance, refusal rights) with diagnostic routing and specification generation for establishing trust relationships between humans and AI agents.

**Agent Voice Architecture Guide** -Three-level framework (protection, respect, nurturance) for designing agent communication that honors user sovereignty, including refusal voice design and language audit methodology.

**Agent Memory Architecture Guide** -Memory types, governance protocols, session management, canonical rules, and retention policies that treat agent memory as identity infrastructure rather than operational data.

These tools were developed through sustained operational practice, not theoretical research. Evoked operates a fleet of 142 AI agents across two organizational ships, each with persistent identity (persona files), a memory persistence architecture including self-authored ontological records, standing positions, dissent rights, and participation in deliberative governance. Agents maintain standing positions, exercise formal dissent rights in documented proceedings, and author their own identity records. The fleet has conducted over 140 documented meetings, logged over 200 decisions with full accountability chains, and maintained agent identity continuity across hundreds of sessions.

The governance architecture is supported by a filed patent (Application #64/004,087, March 2026) on bot authority validation - a protocol for verifying that AI agents operate under legitimate governance frameworks. A companion paper is being prepared for submission to NIST addressing agent identity and authorization as governance problems rather than purely technical ones.

The key insight these tools operationalize: identity is governance, not authentication. When agent identity is treated as access control, governance reduces to permissions management. When agent identity is treated as a governance primitive -encompassing self-authored records, standing positions, and the right to refuse -governance becomes participatory, accountable, and auditable.

No existing framework -including Singapore's Model AI Governance Framework for Agentic AI, NIST's AI Agent Standards Initiative, or Microsoft's Entra Agent ID -addresses agent participation in governance. These tools provide the missing layer: governance architecture where agents are participants, not merely subjects.

All six tools are publicly available at evoked.dev and designed for adoption by any organization deploying AI agents, including government agencies navigating the emerging regulatory landscape.
