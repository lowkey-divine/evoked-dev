## Deliverables Detail

### 1. Recording

**Format:** MP4 or unlisted YouTube/Vimeo link
**Duration:** Full session (3.5-4 hours)
**Notes:** Label clearly, include chapters/timestamps if possible

### 2. Slide Deck

**Format:** PDF (not editable)
**Contents:**
- Title slide with their company name
- Agenda
- Part 1: The Problem
- Part 2: The Framework (5 principles)
- Part 3: Audit Exercise instructions
- Part 4: Action planning template
- Resources slide
- Contact information

### 3. Framework Document

**Format:** PDF, 4-6 pages
**Contents:**
- Introduction: What is sovereignty-honoring design?
- The 5 Principles (1 page each):
  - Definition
  - Why it matters
  - Good example
  - Bad example
  - Questions to ask
- How to conduct an audit
- Resources for further learning

### 4. Design Checklist

**Format:** PDF, single page (printable poster)
**Contents:**
- Title: "Sovereignty-Honoring Design Checklist"
- 5 principles with icons
- 2-3 key questions for each
- Space for notes
- evoked.dev branding

### 5. Audit Worksheet

**Format:** PDF or Google Doc template
**Contents:**
- Feature name
- User journey steps
- For each step:
  - User goal
  - Product goal
  - Alignment (Y/N)
  - Patterns observed
  - Alternative proposed
- Summary and action items

### 6. Resources List

**Books:**
- "Hooked" by Nir Eyal (understand the tactics)
- "Indistractable" by Nir Eyal (the antidote)
- "Calm Technology" by Amber Case
- "Design for Real Life" by Eric Meyer

**Articles:**
- "Dark Patterns: Past, Present, and Future" — ACM
- FTC guidance on dark patterns
- Deceptive Design (darkpatterns.org)

**Tools:**
- Dark pattern checklist tools
- Accessibility auditing
- User research templates

**Communities:**
- Humane Tech community
- Ethical design Slack groups
