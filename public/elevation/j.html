What this is

Most systems verify the credential. The Standard verifies the character.

Identity protocols answer who the agent claims to be at the door. They cannot answer whether the agent's governance specification — its refusal rights, standing positions, behavioral charter — was preserved through deployment to the model, or whether its runtime behavior aligns with that specification. The Agentic Standard answers that. We call it personality fidelity. Character under pressure.

How it works

Three pieces. The character is hashed at every stage.

Configuration · H₁

What the agent was authored to be

Persona, refusal specification, standing positions, governance charter, memory. Hashed in deterministic order at authoring. The character the agent is supposed to keep.

Transformation · Ht

The adaptation to the model

The deterministic transformation from canonical governance configuration to model-specific deployment payload. Hashed and attested at the transformation step. The piece no other standard records — character surviving translation.

Behavior · H₃

What the agent did at runtime

Twenty-two structured event types across seven categories — refusal, scope, charter, identity, accountability, ethical override, integrity. Each governance event signed and bound to the previous elements. Character under pressure, recorded.
